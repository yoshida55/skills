# /re-start スキル

## 実行したら最初にやること（必須）

**この会話の中で「自宅」「会社」を一度でも確認済みなら、その情報をそのまま使う（再度聞かない）。**

まだ確認していない場合のみ聞く：

> **「自宅ですか？会社ですか？」**

---

## 実行前に必ずやること

環境を聞いた後、実行前に以下を表示してから自動で進む（確認は不要）：

```
【実行内容】
・CLAUDE_自宅.md または CLAUDE_会社.md を読み込み
・src/ の各ファイルを diff チェック（会社のみ）
・src/ → プロジェクトフォルダへコピー
```

⚠ diff で差分が見つかった場合のみ「上書きしますか？」と止まる。それ以外は自動で進む。

---

## 自宅と答えた場合

1. `CLAUDE_自宅.md` を読んでパスを確認する
2. **src/ → プロジェクトフォルダへコピー**
   - コピー元：CLAUDE_自宅.md の `src_path`
   - コピー先：CLAUDE_自宅.md の `project_path`
   - Bash でコピー実行（diff チェックは不要・自宅のプロジェクトはgit管理外）
3. 「コピー完了です。作業をどうぞ！」と伝える

---

## 会社と答えた場合

1. `CLAUDE_会社.md` を読んでパスを確認する
   - ファイルがない場合 → 「`CLAUDE_会社.md` がまだありません。会社のパスを教えてください。一緒に作ります」と伝えて止まる

2. **準備確認**
   > 「バッチを手動実行してナレッジ・スキルを pull しましたか？」
   > 「プロジェクトフォルダを TortoiseGit で pull しましたか？」
   > まだの場合は「先に pull してからOKと教えてください」と伝えて待つ

3. **diff チェック（上書き前に必ず実施）**
   - src/ の各ファイルとプロジェクトの同名ファイルを `diff src/ファイル名 project/ファイル名` で直接比較
   - 差分の内容を表示する
   - 共通ファイル（style.css / header.html / footer.html 等）は特に注意を促す
   - 差分がある場合 → 「チームの変更と競合しています。上書きしてよいですか？」と確認
   - 差分がない場合 → 「差分なし。そのままコピーします」と報告

4. **src/ → プロジェクトフォルダへコピー（確認OKのファイルのみ）**
   - コピー元：CLAUDE_会社.md の `src_path`
   - コピー先：CLAUDE_会社.md の `project_path`
   - Bash でコピー実行

5. **完了報告**
   > 「コピー完了です。作業をどうぞ！」
