# auto-com スキル

> ## ⚡ 絶対最初にやること
> **スキル読み込み直後、作業の1行目は必ず `TodoWrite` でStep 0のチェックリストを出す**

完成したHTMLを読み取り、CSSにコメントだけを書くスキル。
CSSのスタイル（プロパティ・値）は一切書かない。

---

## 🎯 やること・やらないこと

| ✅ やること | ❌ やらないこと |
|---|---|
| セクション区切りコメント | `display: flex` 等のスタイル |
| flex/gridのレイアウト注釈 | プロパティの値 |
| 子要素の `└` コメント | CSSルールセット（`{ }` の中身） |
| わからない箇所はユーザーに聞く | 推測で書く |

---

## 実行フロー

### Step 0. TodoWrite（絶対最初に実行）

```
TodoWrite([
  { content: "スクショ・HTMLの確認",             status: "in_progress", activeForm: "スクショ・HTMLを確認中" },
  { content: "HTMLからレイアウト構造を読み取る",  status: "pending",     activeForm: "レイアウト構造を読み取り中" },
  { content: "不明な箇所をユーザーに確認",        status: "pending",     activeForm: "ユーザーに確認中" },
  { content: "CSSにコメントを書き込む",           status: "pending",     activeForm: "CSSにコメントを書き込み中" }
])
```

### Step 1. スクショ・HTML確認

スクショが渡されていない場合：
**「スクリーンショットを貼ってください。レイアウトの判断に使います。」**

HTMLが渡されていない場合：
**「完成したHTMLを貼ってください。」**

### Step 2. レイアウト構造を読み取る

HTMLを見て以下を判断する：
- 横並び → `flex`（row）
- 縦並び → `flex`（column）or 通常フロー
- 格子状 → `grid`
- 重なり・はみ出し → `absolute`
- 判断できない → ユーザーに聞く

### Step 3. 不明な箇所はユーザーに聞く

```
❓ 「.works_inner は横並びですか？縦ですか？」
❓ 「.hero_panel は absolute ですか？flex ですか？」
```

すべて確認が取れたら Step 4 へ。

### Step 4. CSSにコメントを書き込む

対象CSSファイルを確認して追記する。
ファイルが不明な場合はユーザーに聞く。

---

## 📐 コメントフォーマット

### セクション区切り
```css
/* ============================================================
   SECTION NAME
============================================================ */
```

### クラスごとのコメント（日本語のみ）
- クラス名は書かない（すぐ下にあるため）
- 記号（【 】など）は使わない
- 文字数に応じた出し分け：
  - **6文字以内の場合**: `()` で補足を書く（例: `/* タイトル（作業内容） */`）
  - **7文字以上、または説明文など長い場合**: `()` は書かない（例: `/* 説明文 */`, `/* セクションラベル */`）
- 重要なプロパティは `()` 内に併記する（例: `/* 背景（absolute） */`）

---

## ルール

- コメントのみ書く（プロパティ・値は一切書かない）
- クラス名は含めず、日本語のみで簡潔に書く
- 親と子の関係が重要な場合は、段落や補足で表現する
- わからなければ推測せず必ずユーザーに聞く
- セクション区切りコメントは必ず入れる
