## ⚡ Step 1.5. 出題候補を行番号付きで一括取得（高速化）

PC判定後、以下のコマンドで出題候補を**行番号付き**で一括取得してメモリに保持する。
これにより各問採点時の Grep が不要になり、Edit を直接実行できる。

```bash
# ❌ 優先
grep -n "^- ❌" "D:/50_knowledge/00_mis_log.md" | head -20

# マークなし（未出題）
grep -n "^- [^✅❌🎯📋🔍💡⚠\`\*]" "D:/50_knowledge/00_mis_log.md" | grep -v "^[0-9]*:- \`\`\`\|^[0-9]*:- [0-9]\|^[0-9]*:- !" | head -30
```

取得した行番号を記憶して、採点時は **Grep なしで直接 Edit** する。

---

## 📋 Step 2. 出題優先順位

ファイルを読んで以下の順で出題する：

| 優先度 | 条件 | 出題頻度 |
|---|---|---|
| 1位 | 行に `*`（アスタリスク）が含まれる | 毎回・最優先 |
| 2位 | `❌` 不正解（要復習） | 毎回出す |
| 3位 | マークなし（未出題） | 毎回出す |
| 4位 | `✅` 正解（定着済み） | 5問に1回 |

### ⭐ アスタリスク（*）ルール
- 行に `*` が含まれている項目は**最優先で出題する**
- `*` は「特に重要・間違えやすい」の目印として扱う
- `✅` が付いていても `*` があれば3問に1回は出す

---

## 🎯 Step 3. 出題ルール

- **1問ずつ出す**（まとめて出さない）
- **今日の日付セクション（`## YYYY-MM-DD` が今日）の項目は出題しない**（当日追加分はスキップ）
- **同じ会話内で既に出題した問題は出題しない**（短期記憶対策）
- 問題形式は内容に合わせて選ぶ：
  - 穴埋め：`get_the_category() は〇〇で返る`
  - 選択：`A: echo必要 / B: 不要`
  - コード補完：`このコードの〇〇に入るものは？`
- ヒントは求められたとき or 明らかに詰まっているときだけ出す

### 💼 実務活用問題の出し方（重要）
- 単純な知識問題より「**実務ではどこで使うか**」の形式を優先する
- 例：
  - ❌「`get_term_link()` とは何か？」
  - ✅「`get_term_link()` はどのファイルのどんな場面で使いますか？」
- コードが出題内容の場合は「このコードはどのファイルに書く？」を必ず添える
- 選択肢がある場合は「実務でよく使う方はどちら？」の視点で作る

---

## ✅ Step 4. 採点・記録ルール

### 正解の場合
- 対象行の先頭に `✅` を付ける（既に `✅` なら変えない）
- 「正解です！✅」と伝えて次の問題へ

### 不正解の場合
- 対象行の先頭に `❌` を付ける（既に `❌` なら変えない）
- 正解を解説して次の問題へ

### マークの付け方（Edit ツールで既存行を書き換える）
```
- get_the_category() は配列で返る
↓ 正解なら
- ✅ get_the_category() は配列で返る
↓ 不正解なら
- ❌ get_the_category() は配列で返る
```

## 📝 Step 6. クイズ履歴テーブルを更新する

**10問ごとのサマリー後、または「ミスログに書いて」と言われたとき**に必ず実行する。

### ルール
- ファイル冒頭の `## 📊 クイズ履歴` テーブルに1行追記する
- 同じ日付がすでにある場合は `②` `③` を末尾に付けて区別する
- 「残り未出題」は `❌` マークの行 + マークなしの行の合計数を Grep でカウントして記録する
- **累計正解数**：テーブルの全「正解」列を合計して末尾に記録する（「累計：〇〇問正解」）

### テーブルのヘッダー（次回から累計列を追加）
```
| 日付 | 出題数 | 正解 | 残り未出題 | 累計正解 |
|---|---|---|---|---|
```
追記時は累計正解を前回の累計 + 今回の正解数で計算して記入する。

### 残り問題数のカウント方法
```bash
# ❌ の行数
grep -c "^- ❌" D:/50_knowledge/00_mis_log.md

# マークなし（- で始まり ✅/❌ がない行）の概算
grep -c "^- [^✅❌]" D:/50_knowledge/00_mis_log.md
```
両方の合計を「残り未出題」として記録する。

### 書き方（Edit ツールで追記）
```
| 2026-04-20② | 10問 | 6問 | - |
```

テーブルの末尾行を old_string にして、その行 + 新しい行を new_string にして追記する：
```
old_string: | 2026-04-20 | 10問 | 8問 | 154問 |
new_string:
| 2026-04-20 | 10問 | 8問 | 154問 |
| 2026-04-20② | 10問 | 6問 | - |
```

---

## ⚠ 注意事項

- `✅` が付いている項目は基本的にスキップ（5問に1回だけ出す）
- マークは **Edit ツール** で付ける（memo_write は追記専用なので使わない）
- 問題は **1問ずつ** 出す。まとめて出さない

## 🔄 「もう一度」ルール

ユーザーが「もう一度」と言ったら：
1. 直前に `✅` を付けた行から `✅` を外す（Edit ツールで削除）
2. 「✅ を外しました。次回から再出題されます」と伝える
3. 同じ問題をすぐ再出題はしない（次の問題に進む）

---

## ⭐ 「次も出して」ルール（繰り返し出題マーク）

ユーザーが「次も出して」「繰り返して」「重要」などと言ったら：
1. 直前に出題した問題の行末に ` *` を追記する（Edit ツールで）
2. 「次回から優先的に出します ⭐」と伝える
3. `*` が付いた行は最優先ルール対象になるので、次回クイズで自動的に繰り返し出題される

```
- ❌ カテゴリーアーカイブのURL取得 → get_term_link('スラッグ', 'category')
↓「次も出して」と言われたら
- ❌ カテゴリーアーカイブのURL取得 → get_term_link('スラッグ', 'category') *
```

## 🛑 理解確認ルール（絶対厳守）

- 不正解だった問題・「わからない」と言われた問題は、**すぐ次の問題に移動しない**
- 正解を解説した後、必ずユーザーが理解したかを確認する
  - 例：「理解できましたか？　次の問題に進みますか？」
- ユーザーが「わかった」「OK」「次」などと言って初めて次の問題へ進む
- ❌ 解説だけして即次の問題に進む → 禁止
