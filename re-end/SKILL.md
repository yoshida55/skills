# /re-end スキル

## 実行したら最初にやること（必須）

**この会話の中で「自宅」「会社」を一度でも確認済みなら、その情報をそのまま使う（再度聞かない）。**

まだ確認していない場合のみ聞く：

> **「自宅ですか？会社ですか？」**

---

## 実行前に必ずやること

環境を聞いた後、実行前に以下を表示してから自動で進む（確認は不要）：

```
【実行内容】
・CLAUDE_自宅.md または CLAUDE_会社.md を読み込み
・プロジェクトフォルダ → src/ へコピー（*.html *.css *.js *.php のみ・img/等は除外）
・バッチを手動実行して push を促す（自宅・会社両方）
```

⚠ 確認なしで自動実行する。問題が起きた場合のみ止まる。

---

## 自宅と答えた場合

1. `CLAUDE_自宅.md` を読んでパスを確認する
2. **プロジェクトフォルダ → src/ へコピーする**
   - コピー元：CLAUDE_自宅.md の `project_path`
   - コピー先：CLAUDE_自宅.md の `src_path`
   - コピー対象：`*.html` `*.css` `*.js` `*.php`
   - コピー除外：`img/` `fonts/` `images/` 等の画像・アセットフォルダ
   - Bash で上書きコピー実行
3. 完了報告・バッチ実行を促す
   > 「src/ へのコピー完了です。」
   > 「バッチを手動実行してナレッジ・スキルを push してください。」
   > 「push すると会社に同期されます！」

---
