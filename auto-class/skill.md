# auto-class スキル

> ⚡ **最初に必ず `TodoWrite` でチェックリストを出す。それ以外の作業は禁止。**

> 🖱️ **適用範囲**: `<ide_selection>` 内のコードのみ。選択がなければ「選択してください」と伝えて終了。

---

```
TodoWrite([
  { content: "既存HTMLをgrepして共通クラス確認", status: "in_progress", activeForm: "既存クラスをgrep確認中" },
  { content: "スクショのテキストとHTMLのテキストを1行ずつ照合（h1・p・span等）→ 差分があれば報告・修正確認", status: "pending", activeForm: "スクショとテキスト照合中" },
  { content: "全タグにクラス名付与・セマンティックチェック・構造問題の指摘を同時に行う", status: "pending", activeForm: "クラス名付与中" },

```

- スクショがあれば → スクショが正式な情報源
- なければ → HTMLコメントとテキストから判断
- 共通クラスが既存にあれば → 新規クラスを作らず使い回す

## 📐 命名規則

### 基本
- 形式: `[section]_[element]`（2単語が原則）
- 接続: `_` のみ（`-` ハイフン禁止・キャメルケース禁止）
- 外枠: `_area`（`_container` 禁止）
- 番号禁止: `item1` `item2` → `item` を複数回使う
- 省略禁止: `btn` → `button`、`desc` → `description`（`img` `nav` のみOK）

### 3単語のルール
3単語になりそうなとき：
① `.親クラス 子タグ {}` で CSS に書けるならクラス不要
② 役割を直接表す2単語にできないか検討
③ どうしても無理な場合のみ3単語（4単語以上禁止）

### よく使うパターン
| 役割 | クラス名 |
|------|----------|
| 外枠 | `[s]_area` |
| 見出し | `[s]_title` |
| 本文 | `[s]_description` |
| ボタン | `[s]_button` |
| 画像 | `[s]_img` |
| ラベル | `[s]_label` |
| 1項目 | `[s]_item` |

---

## 🚨 構造ルール

- 不要な wrapper div 禁止（Grid/Flex で解決できないか確認）
- 3階層以上のネスト → ユーザーに確認
- 装飾文字（`-` `|`）→ `::before/::after` で書く・HTMLに書かない
- p タグ等の長文は1行で記述（コード上での改行なし）

---

## ⚠️ 確認・指摘ルール

- ネスト深すぎ・文法ミス・役割不明 → ユーザーに1行で確認
- セマンティックタグが不適切 → 「〇〇は `<タグ名>` が適切です」と1行で指摘（修正しない）
  - 住所→`<address>` / 記事→`<article>` / 日付→`<time>` / ナビ→`<nav>`
- 不足タグ（`<main>` `<nav>` `alt` 属性等）→ 提案のみ（追加はユーザー指示後）
- 構造とレイアウト意図が合っていない → 1行で指摘（修正しない）

---

## 注意事項

1. 全タグ・インライン要素（`span` `strong` `em` `a`）にもクラスを付ける
2. 既存クラスは上書きOK
3. HTMLコメントは削除しない
4. 勝手に構造変更しない
5. 明らかな文法ミス（コメント形式等）は自動修正OK
