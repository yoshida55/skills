# 🎓 GitHub Copilot 専用：HTMLクラス付与・設計指示（最低限・シンプル設計）

> **目的**: Claude Code のような「全タグ付与」ではなく、GitHub Copilot では「メンテナンスしやすさ」と「読みやすさ」を最優先し、極限までクラスと階層を削る。

## 🎯 鉄則：引き算の設計（最優先）

### 1. 「全タグへのクラス付与」は禁止
- 子要素（span, a, p, img など）にむやみにクラスを付けない。
- 親要素にクラスがあれば、CSSで `.parent > img` のように指定できるため、クラスは不要。

### 2. 階層を浅く（入れ子の排除）
- 不要な `div` のラッピング（wrapper）を徹底的に排除する。
- 構造をフラットに保ち、インデントが深くならないようにする。

### 3. CSSセレクタの積極活用
- `:first-child`, `:nth-child()`, `+` (隣接セレクタ) 等で指定できる場合は、HTML側に目印（クラス）を置かない。

---

## 📐 命名・構造ルール

### セクション命名（2単語原則）
- 原則: `[section]_area` (外枠), `[section]_list` (グループ)
- 3単語になりそうな時は、必ず親セレクタで代用できないか検討する。

### ブロック設計
- 繰り返し項目（item）にも、親が `list` ならクラスは不要。
- `.list > div` でレイアウトを組む。

### 具体例（Before/After）

❌ **深すぎる構造（Claude Code/Auto-Class 初期設定）**
```html
<div class="news_area">
  <div class="news_list">
    <div class="news_item">
      <img class="news_img" src="..." />
      <p class="news_text">テキスト</p>
    </div>
  </div>
</div>
```

✅ **GitHub Copilot 推奨構造（シンプル・フラット）**
```html
<section class="news_area">
  <div class="news_list">
    <div> <!-- クラス不要 -->
      <img src="..." /> <!-- クラス不要 -->
      <p>テキスト</p>    <!-- クラス不要 -->
    </div>
  </div>
</section>
```

---

## 🚫 禁止事項
- `<span>` や `<a>` への「とりあえずクラス」の付与。
- デザイン上、1つのクラスで一括指定できる箇所を分けて命名すること。
- HTMLを汚す（クラスを増やす）ことを「丁寧」と勘違いしない。

---
## 💡 AIへの指示（実行時）
この指示書を読み込んだ後は、常に**「クラスをどうやって減らすか」**を考えながらHTMLを生成すること。
「深すぎる」とユーザーに言わせない、洗練されたシンプルなコードを追求せよ。
